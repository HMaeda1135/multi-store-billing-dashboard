# 複数店舗 仕入れ・請求管理ダッシュボード

> **本リポジトリは自主制作ポートフォリオ用のサンプルアプリケーションです。**  
> 実在の企業・案件・業界固有情報は一切使用しておらず、架空の小規模事業者を想定したデモデータで動作します。

複数店舗の仕入れ・請求データを一元管理する業務画面の実装力を示すための React / TypeScript 製 Web アプリケーションです。

## 概要

小規模事業者が複数店舗の仕入れ・請求情報をまとめて管理する場面を想定し、以下の業務フローをカバーしています。

- ロール別（管理者 / 店舗担当者）の簡易ログイン
- 管理者向けダッシュボード（集計・推移・最近の登録データ）
- 仕入れ・請求データの一覧・検索・絞り込み・CSV出力
- 仕入れ・請求データの新規登録・編集・詳細確認
- 店舗マスタ・取引先マスタの管理

## 使用技術

| カテゴリ | 技術 |
|---------|------|
| フレームワーク | React 19 |
| 言語 | TypeScript |
| ビルドツール | Vite |
| ルーティング | React Router |
| データ取得 | TanStack Query |
| フォーム | React Hook Form + Zod |
| テスト | Vitest + React Testing Library |
| スタイリング | CSS Modules |
| データ層 | モック API（インメモリ） |

## 画面一覧

| 画面 | パス | 説明 | アクセス |
|------|------|------|----------|
| ログイン | `/login` | 管理者・店舗担当者を選択してログイン | 全員 |
| ダッシュボード | `/dashboard` | 今月の集計・店舗別集計・月別推移 | 管理者のみ |
| 仕入れ・請求一覧 | `/transactions` | 一覧・検索・絞り込み・CSV出力 | 全員 |
| 新規登録 | `/transactions/new` | 仕入れ・請求データの新規登録 | 全員 |
| 詳細 | `/transactions/:id` | 1件の詳細表示 | 全員 |
| 編集 | `/transactions/:id/edit` | データの編集 | 全員 |
| 店舗管理 | `/stores` | 店舗マスタの一覧・追加・編集 | 管理者のみ |
| 取引先管理 | `/vendors` | 取引先マスタの一覧・追加・編集 | 管理者のみ |

### ロール別の違い

- **管理者**: 全店舗のデータ閲覧、ダッシュボード、店舗・取引先マスタ管理
- **店舗担当者**: 自店舗のデータのみ閲覧・登録（マスタ管理・ダッシュボードは非表示）

## 起動方法

### 前提

- Node.js 18 以上

### インストール

```bash
npm install
```

### 開発サーバー起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開き、ログイン画面からユーザーを選択してください。

### ビルド

```bash
npm run build
```

### テスト

```bash
npm test
```

## プロジェクト構成

```
src/
├── api/           # API関数（モック、将来REST APIに差し替え可能）
├── components/    # 共通コンポーネント・レイアウト
├── contexts/      # 認証コンテキスト
├── hooks/         # TanStack Query フック
├── lib/           # ユーティリティ（フォーマット、CSV、ラベル）
├── mock/          # モックデータ・インメモリストレージ
├── pages/         # 画面コンポーネント
├── routes/        # ルーティング・認可
├── schemas/       # Zodバリデーションスキーマ
├── styles/        # グローバルCSS
└── types/         # 型定義
```
